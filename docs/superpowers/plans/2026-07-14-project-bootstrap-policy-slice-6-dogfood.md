# Project Bootstrap Policy Slice 6 Dogfood Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Bootstrap agentharness itself, prove packed-artifact installation and a real protected PR lifecycle, measure usability/performance, and publish complete user/operator/plugin documentation.

**Architecture:** The repository becomes its first production consumer through a committed `.agentharness-policy/` profile generated by the same packaged artifact users receive. Existing checks remain requirements. Acceptance tooling reads a machine-readable ledger, runs clean fixtures, and refuses release until every criterion has reproducible evidence.

**Tech Stack:** The completed agentharness CLI, npm pack, disposable containers/directories, GitHub Actions, pytest/Bats, mutmut, Markdown documentation, and existing manifest/client generators.
